## Omada Upgrade to 6.2.0.12
Long/painful upgrade process. Key lessons:
- Use `DEBIAN_FRONTEND=noninteractive TERM=xterm dpkg -i` for non-interactive install
- The upgrade reset cluster config: old IPs in `omada_hsb_cluster.properties` (192.168.0.x → should be 10.3.10.9/10)
- The upgrade also commented out all cluster settings in `omada.properties`
- **haKeyFile permissions bug**: upgrade left haKeyFile as 666 on main node — MongoDB refuses to start with world-readable keyfile, needs 400 owned by omada
- **Cluster recovery procedure** (for future reference):
  1. Fix `omada_hsb_cluster.properties` on both nodes with correct IPs
  2. Uncomment `failover.cluster.enable=true`, `mongo.node.ip`, `mongo.node.remote.ip` in `omada.properties` on both nodes (correct IPs for each)
  3. Fix `haKeyFile` perms: `chmod 400 /opt/tplink/EAPController/data/cluster/haKeyFile`
  4. On PRIMARY only: uncomment `mongo.node.start.init.enable=true`
  5. Reboot backup LXC first, then primary LXC
  6. Wait for gRPC :9098 to stay up on both nodes
  7. Re-comment `mongo.node.start.init.enable=true` on primary
- gRPC port for cluster inter-node: **9098**
- MongoDB replSet name: **omadaCluster_8507**
