## Concepts clarified for Miguel
- `session.maintenance` = sessions.json housekeeping, NOT memory files
- Compaction IS trimming (smarter form of it), not avoiding it
- Memory flush fires BEFORE compaction to write durable notes first
- `safeguard` mode = least expensive for long-running sessions, fewer compaction passes
- Miguel's profile: few sessions, long-running Slack channels → safeguard + cache-ttl pruning is optimal
- `/compress` is not a native command — correct command is `/compact`
