## Tools & Patterns Learned (This Session)

### mcporter — Large Payload Pattern
**Issue:** exec runner does not invoke shell, so `$(cat ...)` subshell expansion fails silently. Cannot inline 16-18KB of HTML content directly.

**Solution that works:**
1. Write payload to JSON file in tmp/
2. Spawn subagent (they have shell access)
3. Subagent runs: `/opt/homebrew/bin/mcporter call server.tool --args "$(cat /tmp/payload.json)"`
4. Subagent handles the shell expansion

**Why:** Subagents run in full shell environment. Main agent exec runner passes commands directly to process without shell interpretation.

**Documented in:** TOOLS.md section "mcporter — Large Payload Pattern"

### Confluence Workflow Gotchas
- `confluence_move_page` requires `--args '{"page_id":"...","target_parent_id":"..."}'` — plain key=value breaks (mcporter converts ints, tool expects strings)
- `confluence_update_page` with page_id=<int> gets type error; need `--args` with proper JSON
- Short inline content: function call syntax works fine
- Long content (>1KB): must use subagent + JSON payload file pattern
